### On a :

    - Un client qui a un site Wordpress
    - Un site qui est décoré d'un thème
    
### Problèmes :
        
    - Https (à mettre en place)
    - SSL (demander ce que c'est et quel(s) problème(s) rencontré?)
    - La version de PHP pas à jour
    - Site inaccessible
    - Le thème rendait le site inaccessible

### Possibilités :

    - Garder le thème actuel et essayer de rendre le site accessible
    - Changer le thème mais le site est accessible

### Extensions :

        #### limiter les extensions car elles surchargent le haut de page

    - Protectyouradmin non sécurisée => application de sécurité
    - WordFence : extension de sécurité => s'orienter sur Secupress (Extension de sécurité très complète)
    - Wprocket
    - Updraftplus : ne pas parametrer en local mais en sauvegarde drive 
    - Yoast SEO 
    - Google Analytics
    - Intégrer WPML(Wordpressmultilingual)?
---

- Sauvegarder les fichiers pour avoir une sorte de protection, éviter de sauvegarder en local pour Updraftplus car il va y avoir une surcharge sur le site
qui peut mener a une inaccessibilité au site.

- Dark mode / Light mode => A la fin, quand tout est posé, thème choisi etc.

- S'orienter sur Astra comme thème, assez flexible, facile d'utilisation, assez simple, regulièrement mis à jour.

- Fluxus : Version utilisée par le client 1.5.1 
Version Actuelle : 1.9 (futur updates)
Est-ce qu'il faut réacheter une license pour chaque update avant la 1.9 ? 
Dans tout les cas le client peut reprendre une license Fluxus pour téléverser tout ce qu'il avait fait tout en gardant le contenu actuel de son site WP
